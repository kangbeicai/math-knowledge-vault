# 笔记生命周期

## 1. 捕获

临时内容先进入 `00 Inbox/`。不要求完整，但应记录触发来源和下一步动作。

## 2. 分类

判断内容属于概念、问题、来源、练习、项目还是复盘。无法判断时继续留在 Inbox。

## 3. 原子化

长文章不直接变成一篇巨型笔记，而拆成：

- 一篇来源笔记；
- 多篇概念笔记；
- 若干问题笔记；
- 一组练习；
- 一个 MOC 或学习路径入口。

## 4. 连接

正式概念笔记至少应：

- 链接到一个 MOC；
- 声明必要前置概念；
- 记录主要来源；
- 指向例子、问题、练习或项目中的至少一类。

## 5. 验证

概念成熟度依次为：

```text
初见 → 可复述 → 可计算 → 可证明 → 可应用
```

发现错误时改为 `待修正`，保留理解演化记录。

## 6. 复盘

周期复盘检查：

- 长期未整理的 Inbox；
- 只有定义、没有例子或反例的概念；
- 已回答但未回链的问题；
- 与实际掌握程度不符的状态；
- 需要调整的学习路径。

## 7. 归档

不再维护但仍有历史价值的内容使用 `status: archived`，并指明替代笔记。
