# 元数据规范

## 通用结构

```yaml
---
type: concept
area: differential-geometry
status: 初见
created: 2026-06-17
updated: 2026-06-17
aliases: []
prerequisites: []
related: []
sources: []
tags:
  - concept
---
```

## 必需属性

| 属性 | 含义 | 示例 |
|---|---|---|
| `type` | 笔记类型 | `concept` |
| `status` | 成熟度或工作状态 | `可复述` |
| `tags` | 跨目录检索标签 | `concept` |

概念、MOC、问题、练习和项目还应包含 `area`。

## 推荐属性

| 属性 | 含义 |
|---|---|
| `created` | 首次创建日期 |
| `updated` | 最近一次实质更新日期 |
| `aliases` | 英文名、别名和常见译名 |
| `prerequisites` | 必要前置概念 |
| `related` | 非前置的相关概念 |
| `sources` | 主要来源笔记 |

## type 允许值

`home`、`meta`、`moc`、`index`、`concept`、`learning-path`、`question`、`exercise-set`、`project`、`source`、`review`、`dashboard`、`inbox`。

## area 建议值

`mathematics`、`foundations`、`linear-algebra`、`calculus`、`analysis`、`algebra`、`topology`、`geometry-topology`、`differential-geometry`、`probability-statistics`、`differential-equations`、`numerical-mathematics`、`optimization`、`mathematical-physics`、`interdisciplinary`。

## status 规范

### 概念成熟度

`初见` → `可复述` → `可计算` → `可证明` → `可应用`

发现错误时使用 `待修正`。

### 工作流状态

`seed`、`active`、`open`、`answered`、`todo`、`doing`、`done`、`reading`、`unread`、`archived`。

## 属性命名

- 使用小写英文和下划线，例如 `source_type`、`source_map`。
- 避免同一概念出现多种拼写。
- 链接属性使用带引号的字符串列表：

```yaml
prerequisites:
  - "[[向量空间与线性映射]]"
  - "[[多元链式法则与Jacobian]]"
```
