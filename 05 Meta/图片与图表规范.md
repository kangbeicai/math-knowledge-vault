# 图片与图表规范

## 1. 图片的职责

图片用于表达文字和公式不容易直接呈现的内容，例如：

- 空间中的相对位置；
- 局部与整体的差异；
- 坐标图之间的覆盖和转换；
- 切空间、法向和映射方向；
- 随时间变化的流；
- 积分区域、边界与定向。

图片不应替代：

- 正式定义；
- 必要的公式推导；
- 条件说明；
- 对图中符号的正文解释。

## 2. 何时值得配图？

满足以下任一条件时，优先考虑图片：

- 需要同时理解多个空间或映射；
- 概念依赖局部与整体的对比；
- 文字描述容易产生错误空间直觉；
- 坐标奇点、覆盖关系或方向关系难以仅靠公式表达；
- 动态过程需要展示轨迹或变形。

以下情况通常不需要图片：

- 单纯给出代数定义；
- 一行公式已经足够清楚；
- 图片只是装饰，与理解无关；
- 图像会暗示错误的维数或嵌入方式。

## 3. 目录规范

```text
99 Assets/
├── Images/
│   ├── Manifolds/
│   ├── Linear-Algebra/
│   ├── Analysis/
│   └── Numerical-Methods/
├── Diagrams/
└── PDFs/
```

- `Images/` 保存最终嵌入笔记的文件；
- `Diagrams/` 保存可编辑源文件；
- `PDFs/` 只保存版权和体积均适合进入仓库的附件。

## 4. 文件格式

| 类型 | 推荐格式 | 说明 |
|---|---|---|
| 几何线图、坐标图、流程图 | SVG | 可缩放，首选 |
| 数值曲线、热图 | PNG | 避免有损压缩 |
| 照片或复杂纹理图 | WebP 或 JPG | 控制体积 |
| 动态演示 | GIF 或短视频 | 谨慎使用，避免仓库膨胀 |
| 可编辑图源 | Excalidraw、Draw.io、Canvas | 放入 `Diagrams/` |

## 5. 命名规范

推荐使用英文短文件名，主题前缀明确：

```text
manifold-local-chart.svg
manifold-tangent-space.svg
manifold-pushforward.svg
manifold-stokes-orientation.svg
so3-local-parameterization.svg
```

避免：

- `截图1.png`；
- `未命名.png`；
- 只用日期或随机字符；
- 同一图片保存多个无法区分的副本。

## 6. 在笔记中的嵌入

### Obsidian 内部笔记

```markdown
![[manifold-local-chart.svg|720]]
```

### 需要 GitHub 直接渲染的页面

使用标准 Markdown 相对路径：

```markdown
![流形局部坐标图](../../99%20Assets/Images/Manifolds/manifold-local-chart.svg)
```

README、贡献指南和对外公开说明优先使用标准 Markdown 语法。

## 7. 图片周围必须包含什么？

每张教学图至少应包含：

1. 图前一句话说明读图目的；
2. 有意义的替代文本或标题；
3. 图后解释关键箭头、区域和符号；
4. 必要时说明图不是按比例绘制；
5. 来源或生成方式。

示例结构：

```markdown
下图展示坐标图如何把流形的一小块映射到欧氏开集。

![[manifold-local-chart.svg|720]]

图中的 $U\subset M$ 是流形上的开集，$\varphi(U)\subset\mathbb R^n$ 是其坐标像；箭头表示坐标映射，而不是把流形物理压平。
```

## 8. 来源与版权

- 自制图注明“自制”；
- 根据文献重绘时记录来源并说明“据某来源重绘”；
- 不直接复制来源不明或许可不明的图片；
- 外部图片的来源记录在对应来源笔记中；
- 论文图若仅用于私人研究，也应保留完整出处，不应默认公开再分发。

## 9. 图像质量

- 文字在常用缩放下应清晰；
- 数学符号与正文记号一致；
- 不依赖颜色作为唯一信息通道；
- 黑白打印时仍应基本可辨；
- 线条、箭头和标签不过度拥挤；
- 三维透视图必须避免把环境维数误当作流形维数。

## 10. Git 与文件体积

- 单张教学图尽量小于 1 MB；
- SVG 和压缩后的 PNG 优先；
- 大型原始工程文件谨慎提交；
- 不把临时导出文件和多个重复版本同时纳入 Git；
- 删除附件前先搜索反向链接。

## 11. 流形主题的图示原则

流形示意图应优先表达：

- 对象与坐标表示的区别；
- 环境空间与内在维数的区别；
- 局部坐标覆盖而非全局压平；
- 不同点具有不同切空间；
- 映射如何诱导切向量和微分形式的变换；
- 边界、定向和积分区域之间的关系。

具体制作顺序见 [[流形学习示意图规划]]。

- [ ] 颜色不是唯一信息通道
