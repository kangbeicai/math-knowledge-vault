# 数学书写规范

## 公式分隔符

### 行内公式

```markdown
函数 $f:M\to\mathbb R$ 在点 $p$ 的微分记为 $df_p$。
```

### 独立公式

```markdown
$$
df_p:T_pM\to\mathbb R.
$$
```

为保证 Obsidian 与 GitHub 显示一致，不使用旧式反斜杠圆括号或方括号作为公式分隔符。

## 记号原则

- 定义优先使用 `:=`。
- 普通相等使用 `=`。
- 恒等或同余等特殊关系才使用 `\equiv`。
- 新符号首次出现时说明所属空间、定义域或值域。
- 区分数学对象与坐标、矩阵、分量或离散表示。

## 推导原则

- 公式前说明它为什么出现。
- 公式后解释新符号和成立条件。
- 长推导分步写，不在一行堆叠过多等号。
- 短笔记一般不手工编号；需要引用时使用 `\tag{}`。
